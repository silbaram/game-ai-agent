# 게임 디자인 시스템

이 스킬은 흩어진 영감 자료를 다른 디자이너나 엔지니어가 실제로 구현할 수 있는 게임 UI 시스템으로 정리할 때 사용한다. 목표는 레퍼런스를 그대로 모방하는 것이 아니라, 대상 게임에 맞는 재사용 가능한 규칙, 토큰, 컴포넌트 동작을 추출하는 것이다.

## 작업 방식

- 실제 게임 맥락부터 파악한다: 장르, 플랫폼, 카메라 거리, 템포, 타깃 유저, 수익 모델, 그리고 시스템 범위가 게임플레이 HUD인지 메타 UI인지 둘 다인지 먼저 정리한다.
- 이미지, URL, 설명을 모두 증거로 다룬다. 직접 관찰한 사실과 해석을 분리하고, 추론이 들어간 부분은 명확히 표시한다.
- 우선순위는 게임플레이 가독성, 제품 일관성, 스타일 차별화 순으로 둔다.
- `premium`, `immersive`, `arcade`, `gritty` 같은 추상 형용사는 반드시 구체적인 시각 규칙과 상호작용 규칙으로 번역한다.
- 레퍼런스가 충돌하면 게임 속도, 입력 방식, 시청 거리, 정보 밀도, 플랫폼 제약을 기준으로 우선순위를 정한다.

## 워크플로

1. 브리프를 정규화한다
- 사용자가 원하는 결과가 컨셉 방향 제안인지, 실무 투입 가능한 시스템인지, 기존 시스템의 점검/리팩터링인지 먼저 판별한다.
- 시스템 결정에 실제로 영향을 주는 최소 제약 조건만 우선 수집한다.
- 입력 체크리스트와 증거 정리 방식은 `references/reference-intake.md`를 참고한다.

2. 레퍼런스에서 신호를 추출한다
- 각 레퍼런스를 무드, 계층, 밀도, 형태 언어, 색상 운용, 타이포그래피, 어포던스, 모션, 플랫폼 단서로 분해한다.
- 무엇을 유지할지, 어떻게 변형할지, 무엇을 버릴지 결정한다. 디테일을 그대로 상속하지 않는다.
- URL을 직접 확인할 수 없으면 스크린샷이나 핵심 발췌를 요청한다.

3. 시스템 기초를 정의한다
- 시스템이 왜 그런 형태와 동작을 갖는지 설명하는 디자인 원칙 3-5개를 작성한다.
- 색상, 타이포그래피, 간격, 크기, 반경, 스트로크, 입체감, 아이콘 스케일, 모션 토큰을 정의한다.
- 영감 자료를 기반 규칙으로 전환할 때는 `references/game-ui-foundations.md`를 참고한다.

4. 게임플레이와 메타 화면을 맵핑한다
- 전투 HUD, 내비게이션, 인벤토리, 성장, 상점, 소셜/시스템 UI, 온보딩, 플랫폼 오버레이 등 실제로 필요한 화면을 빠짐없이 다룬다.
- 핵심 컴포넌트나 예외 상태를 놓치지 않도록 `references/component-inventory.md`를 사용한다.

5. 상호작용 동작을 명세한다
- 기본, hover, focus, pressed, selected, disabled, warning, error, loading, success, controller focus 상태를 필요한 범위에서 정의한다.
- 게임이 지원하는 입력 방식에 맞춰 마우스/키보드, 컨트롤러, 터치별 지침을 포함한다.
- 정적인 스타일만 쓰지 말고, 피드백의 타이밍과 의도까지 설명한다.

6. 산출물을 정리한다
- 최종 응답 구조는 `references/output-blueprint.md`를 따른다.
- 사용자가 바로 복사해 쓸 수 있는 명세를 원하면 `assets/design-system-spec-template.md`를 시작점으로 삼는다.
- 마지막에는 Figma 변수, CSS/디자인 토큰, Unity UI Toolkit/uGUI, Unreal UMG/CommonUI 등 요청된 타깃에 맞춘 구현 가이드를 붙인다.

## 출력 기준

- 토큰과 컴포넌트 목록에 들어가기 전에 짧은 시스템 논지부터 제시한다.
- 토큰은 숫자보다 의미를 먼저 정의한다. 각 토큰 군의 역할을 설명한 뒤 대표 값을 제시한다.
- 토큰 정의, 컴포넌트 상태, 플랫폼별 오버라이드는 표 형식을 우선한다.
- `레퍼런스에서 가져온 영감`과 `정식 시스템 규칙`을 분리해서 적는다.
- 실제 구현에 영향을 주는 제약, 실패 패턴, 안티패턴은 반드시 드러낸다.

## 최소 산출물

- 디자인 원칙
- 기반 토큰
- 화면 맵과 컴포넌트 카탈로그
- 상태 및 상호작용 규칙
- 가독성 및 접근성 메모
- 구현 핸드오프 메모
