# 게임 UI 기반 원칙

레퍼런스 분석 이후 시스템의 정식 규칙을 정의할 때 이 파일을 사용한다.

## 디자인 원칙

디자인 의사결정을 실제로 제한할 수 있을 만큼 구체적인 원칙 3-5개를 작성한다. 좋은 원칙은 측정 가능하다.

- `장식보다 전투 가독성을 우선한다`
- `거실 거리에서도 큰 숫자가 바로 읽혀야 한다`
- `고가치 보상은 시끄럽지 않고 의식적이어야 한다`

`현대적이고 세련됨` 같은 비어 있는 문구는 피한다.

## 토큰 계층

토큰은 의미를 먼저 정의하고, 필요할 때 수치로 구체화한다.

### 색상

값보다 역할을 먼저 정한다.

- 캔버스/배경
- 표면/하위 표면
- 주요 액션
- 보조 액션
- 정보 강조
- 긍정, 경고, 위험
- 비활성 또는 잠금
- 희귀도 또는 성장 단계

게임 UI에 특화된 규칙은 다음과 같다.

- 중요한 의미를 색상만으로 전달하지 않는다. 형태, 아이콘, 라벨, 모션 중 하나 이상을 함께 쓴다.
- 전투 경고 색상과 성장/경제 색상은 분리한다.
- 오버레이 UI가 다양한 게임 배경 위에서도 읽히는지 확인한다.

### 타이포그래피

타이포그래피는 크기 단계가 아니라 사용 맥락 기준으로 나눈다.

- 디스플레이 또는 연출용 헤더
- 섹션 헤더
- UI 라벨
- 본문 또는 설명 텍스트
- 마이크로 라벨 또는 메타데이터
- 타이머, 재화, 대미지, 쿨다운용 숫자 강조
- 키/버튼 입력 글리프 조합

다음 항목을 점검한다.

- 숫자에 별도 스타일이 필요한지
- 독일어/프랑스어의 긴 문자열과 CJK 문자열을 견디는지
- 작은 HUD 크기에서 올캡스가 가독성을 해치지 않는지

### 간격과 스케일

- 정보 밀도와 여백을 모두 감당할 수 있는 예측 가능한 공간 스케일을 사용한다.
- 최소 클릭/터치 영역과 컨트롤러 포커스 간격을 정의한다.
- TV와 핸드헬드 환경의 세이프 존과 가장자리 배치를 고려한다.

### 형태 언어

하나의 지배적인 기하 체계를 선택한다.

- 날카롭고 전술적인 형태
- 둥글고 친근한 형태
- 베벨 기반의 고급형 형태
- 다이에제틱 또는 소재 기반 형태

패널, 버튼, 태그, 체력 바, 초상화, 모달에 일관되게 적용한다.

### 스트로크, 입체감, 분리

- UI 표면이 게임플레이와 서로 다른 표면으로부터 어떻게 분리되는지 정의한다.
- 스트로크, 내부 광원, 그림자, 블러, 배경 처리 방식은 의도를 갖고 사용한다.
- 모든 효과를 한꺼번에 겹치지 말고, 소수의 분리 패턴만 정한다.

### 아이코노그래피

- 채움형과 윤곽형의 비중을 정한다.
- 모서리 날카로움, 내부 디테일 수준, 상징형인지 묘사형인지 정의한다.
- 아이콘 스타일을 입력 힌트, 상태 배지, 버프/디버프, 인벤토리 카테고리와 맞춘다.

### 모션과 피드백

모션도 시스템으로 정의한다.

- 일반 전환: 패널 진입, 탭 전환, 화면 전환
- 입력 피드백: hover, focus, click, confirm, error, deny
- 보상 순간: 전리품, 레벨업, 업적, 해금
- 전투 피드백: 적중 확인, 쿨다운 준비 완료, 경고 펄스

실용적인 타이밍 기준은 다음과 같다.

- 미세 피드백: 대략 80-180ms
- 일반 UI 전환: 대략 180-300ms
- 보상 연출 비트: 대략 300-700ms

## 가독성 규칙

- 핵심 게임플레이 정보는 한눈에 읽혀야 한다.
- 가장 강한 대비는 가장 시간 민감한 정보에만 쓴다.
- 전투 HUD 같은 고압 화면에서는 장식 잡음을 줄인다.
- 밝은 장면, 어두운 장면, 움직임이 많은 장면 위에서 UI를 테스트한다.

## 접근성 규칙

- 색상 기반 상태는 아이콘, 텍스트, 형태로 보강한다.
- 컨트롤러와 키보드 탐색에서 포커스 가시성을 유지한다.
- 제품 요구에 따라 텍스트 크기 조절을 지원한다.
- 구조를 블러, bloom, 저대비 반투명 효과에만 의존하지 않는다.
