# 컴포넌트 인벤토리

시스템 범위를 정하고 중요한 게임 UI 화면을 빠뜨리지 않기 위해 이 파일을 사용한다.

## 컴포넌트 명세 최소 항목

각 컴포넌트마다 다음 항목을 정의한다.

- 목적
- 우선순위
- 콘텐츠 모델
- 변형
- 필수 상태
- 입력 동작
- 레이아웃 규칙
- 실패 또는 예외 케이스

## 게임플레이 HUD

- 체력, 실드, 스태미나, 마나, 기타 자원 바
- 스킬 트레이, 쿨다운 표시, 단축키
- 조준점 또는 타깃 인디케이터
- 버프, 디버프, 임시 상태 스트립
- 목표 추적기와 웨이포인트 마커
- 파티 또는 분대 프레임
- 미니맵, 나침반, 내비게이션 리본
- 대미지, 회복, 점수 피드백
- 상황별 프롬프트와 상호작용 힌트
- 보스전, 인카운터, 미션 상태 모듈

## 메타 UI와 내비게이션

- 메인 내비게이션 셸
- 페이지 탭과 서브 내비게이션
- 모달, 드로어, 확인 패턴
- 토스트, 알림, 인박스, 공지 화면
- 프로필, 계정, 세션 상태 UI
- 검색, 필터, 정렬, 페이지네이션 패턴

## 성장과 경제

- 인벤토리 그리드 또는 리스트
- 아이템 카드, 툴팁, 비교 뷰
- 장비 슬롯과 로드아웃 편집기
- 스킬 트리 또는 퍼크 그래프
- 퀘스트, 미션, 저널 뷰
- 상점, 오퍼 카드, 번들 뷰
- 재화 잔액과 프리미엄 액션 요소
- 배틀패스, 이벤트 트랙, 보상 사다리

## 소셜과 시스템

- 친구 목록과 파티 로비
- 채팅, 감정표현, 핑, 기타 커뮤니케이션 요소
- 매치메이킹 또는 대기열 상태
- 설정, 조작, 접근성 패널
- 오류 복구와 재접속 화면

## 플랫폼별 오버라이드

활성 입력 모델 기준으로 모든 핵심 컴포넌트를 점검한다.

- 마우스/키보드: hover 상태, 커서 목표 크기, 단축키 가시성
- 컨트롤러: 포커스 링, 범퍼 내비게이션, 스냅 로직, hover 의존도 축소
- 터치: 터치 영역, 제스처 충돌, 엄지 가동 범위, 가상 키보드 오버레이

## 고밀도 화면

고밀도 게임 UI는 모든 컴포넌트가 눈에 띄려 해서 실패하는 경우가 많다. 아래 항목을 명시적으로 결정한다.

- 어떤 컴포넌트가 포커스 전까지 배경적으로 머물러야 하는지
- 어떤 숫자가 항상 전면에 남아야 하는지
- 어떤 모듈이 접히거나 숨어도 되는지
- 어떤 컴포넌트가 compact/expanded 변형을 가져야 하는지
