커널 터미널 디자인 시스템

터미널의 감각을 유지하면서도 현대적이고 읽기 쉬운 인터페이스를 만들기 위한 디자인 시스템.

커널(Kernel) 은 가상의 "개발자 도구" 브랜드입니다. CLI 에이전트, TUI 대시보드, 문서 사이트까지 아우르는 터미널 중심의 제품군을 가정합니다. 실제 제품이 없는 상태에서 시작된 시스템이므로, 자리표시자 컨셉으로 커널 을 사용합니다. 언제든 다른 브랜드명으로 교체 가능합니다.

제공된 소스

  • 사용자가 별도 코드베이스 / 피그마 / 스크린샷을 제공하지 않음
  • 기본 방향: "터미널 UI/UX 디자인 시스템 — 심플하면서도 터미널 감각을 유지"
  • 기본값으로 진행한 결정:

- 미감: 모던 다크 + 클래식 그린 / 앰버 액센트

- 메인 폰트: JetBrains Mono (모노스페이스, 라틴) + MaruBuri Light (한글, 로커로 탁지)

- 스펙트럼: 터미널 영감형 — 모노스페이스, 프롬프트, 커서 모티프를 쓰되 필요시 모던 UI 요소도 허용

- 제품군: (1) 터미널 TUI 앱, (2) 마케팅 / 문서 웹사이트

폰트 소스
- JetBrains Mono — 구글 폰트 CDN 에서 로드 (라틴 / 숫자 / 기호 전용)
- MaruBuri Light — fonts/MaruBuri-Light.ttf 로컬 파일. 한글 글리프만 커버하도록 unicode-range 로 제한 (라틴은 JetBrains Mono 사용).

콘텐츠 원칙 (CONTENT FUNDAMENTALS)

전체 보이스

간결하고, 기술적이고, 조용히 자신감 있게. man page 와 잘 쓴 README 사이 어딘가. 낭비 없음. 감탄사 없음. 이모지 없음. 스크립트 자체가 이미 재미있으니 불필요한 장식은 오히려 방해가 됩니다.

톤 레인지

  • CLI / TUI: man page 스타일. Usage:, Error:, warning:. 주어 생략 가능. 명령형.
  • 문서: 중립적 기술 문서. "당신" 혹은 "you" 를 허용하되 과하지 않게.
  • 마케팅: 기술자에게 기술자로 말하기. 과장 금지. 구체적 숫자와 동사.

대소문자 규칙

  • 제품명: Kernel (처음 한 번 Pascal, 이후 소문자 kernel 도 허용 — 터미널 명령처럼 느껴지게)
  • 문서 헤더: 문장식 (Sentence case). 절대 Title Case 아님.
  • CLI 출력: 소문자 + 콜론. error: missing argument
  • 상태 코드: OK, FAIL, WARN 같은 대문자 상태 코드는 허용 — 터미널 출력의 일부.

1인칭 vs 2인칭

  • 도구가 1인칭으로 말하지 않음. "I'll help you..." 같은 표현 금지.
  • 문서에서 "당신" / "you" 는 허용.
  • CLI 출력은 완전 중립: saved 3 files in 42ms

이모지

사용 안 함. 상태 표시에는 [OK], [✓], [✗], ● 같은 유니코드 심볼 또는 ASCII 를 사용합니다.

한글 톤

  • 반말 / 존댓말 혼용 금지. 기술 문서는 "~합니다" 체로 통일.
  • 외래어 최소화. 단, 고유명사(커밋, 브랜치 등) 는 그대로.
  • 예시: "안녕하세요! 지금 빌드를 시작할게요 😊" → "빌드를 시작합니다. 3개 파일, 42ms."

실제 카피 예시

# CLI
$ kernel run build
→ compiling 3 modules...
→ ok (42ms)

$ kernel deploy
error: no target configured
  hint: run `kernel init` first

# 마케팅 카피
A terminal for people who ship.
No dashboards. No modals. Just pipes.

# 문서 헤더

비주얼 원칙 (VISUAL FOUNDATIONS)

컬러 시스템

다크 우선. 거의 모든 UI 가 다크 배경 위에서 시작합니다. 라이트 테마는 존재하지만 보조적입니다.

  • 기본 배경: 거의 검정 #0B0D0E (순수 #000 은 피함 — 너무 강함), 패널은 한 단계 위 #13161A
  • 전경: #E6E8EA (본문), #9AA3AD (보조), #5E6670 (희미함), #2A2F36 (거의 보임)
  • 액센트: 포스퍼 그린 #00E26B — CTA, 프롬프트, 포커스. 과하게 쓰지 않음.
  • 의미 컬러: 그린 #00E26B / 앰버 #FFB000 / 레드 #FF4D4D / 시안 #5DD3FF — 터미널 ANSI 색과 정렬

전체 컬러 팔레트는 colors_and_type.css 참조.

타이포그래피

  • 모노스페이스 (라틴): JetBrains Mono — 모든 코드, CLI 출력, UI 크롬, 데이터, 숫자
  • 한글: MaruBuri Light — 본문, 카피, 문서 롱폼, 한글 UI. fonts/MaruBuri-Light.ttf 로컬 파일, unicode-range 로 한글 글리프만 커버.
  • 산세리프 (옵션): IBM Plex Sans — 라틴 롱폼 본문이 모노와 어울리지 않을 때만.
  • 메인 UI 크롬은 모노 기본. 제목도 모노. 한글 본문만 MaruBuri 로 전환.
  • 크기 스케일: 12 / 13 / 14 / 16 / 20 / 28 / 40 (12 가 기본 터미널 크기, 14가 UI 기본)
  • 줄 높이: 모노 1.5 고정. MaruBuri 본문은 1.7 (한글은 좀 더 넓게).

CSS 변수

  • --font-mono — JetBrains Mono 스택
  • --font-kr — MaruBuri (한글 전용)
  • --font-kr-mono — 라틴은 mono, 한글 fallback 으로 MaruBuri
  • --font-sans — IBM Plex Sans (산세리프 필요 시)

간격 (Spacing)

8px 베이스가 아니라 4px 베이스. 터미널 셀이 좁기 때문입니다.

스텝: 2 4 6 8 12 16 24 32 48 64

터미널 셀 너비: 7.2px (14px 모노 기준). 레이아웃은 컬럼 그리드가 아니라 캐릭터 그리드 로 사고합니다.

테두리 (Borders)

  • 모든 기본 테두리는 1px solid — 더 두껍지 않음.
  • 색: #2A2F36 (희미), #3D434D (기본), #00E26B (포커스)
  • 둥근 모서리: 0 또는 2px. 4px 이상 금지. 터미널은 각져있습니다.
  • ASCII 박스 드로잉 (─ │ ╭ ╮ ╰ ╯) 을 장식적으로 사용 허용.

그림자 (Shadows)

기본은 그림자 없음. 터미널은 평면입니다.

허용되는 유일한 섀도는 글로우(glow) — 포커스된 요소에 0 0 0 1px accent, 0 0 12px accent/20%. 드물게 사용.

모서리 반경 (Corners & radii)

  • 컴포넌트: 0px 또는 2px
  • 큰 카드: 최대 4px
  • 원형 / pill: 금지 (배지조차도 사각형)

배경 (Backgrounds)

  • 주 배경: 단색 #0B0D0E
  • 보조 배경 옵션: 은은한 스캔라인 (1px 간격, 2% 불투명도) — 극히 드물게, 히어로에만
  • 그라디언트 금지 (특히 blue→purple 같은 AI 슬롭 류). 단, 터미널 하단 페이드 (아래쪽으로 검정) 는 허용
  • 이미지: 거의 사용 안 함. 썼다면 채도를 낮추고 그린 틴트 또는 흑백.

애니메이션

  • 짧고 기능적. 100–150ms. 감속 커브가 아닌 steps(1) 또는 linear 선호 (터미널처럼 뚜렷하게 딱딱).
  • 핵심 모티프: 깜빡이는 커서 (블록 커서, 1초 주기)
  • 타이핑 애니메이션: 실제 타자기처럼 글자씩 등장 가능
  • 페이드 허용하되 바운스 금지. 스프링 금지.
  • 호버: opacity 0.9 → 1.0 혹은 배경 +4% 밝기. 확대 금지.
  • 프레스: 배경 -4% 밝기. 크기 변화 금지.

포커스 / 선택

  • 모든 포커스 링은 액센트 그린 1px solid 바깥쪽 + 0 outline
  • 선택 배경: 액센트 그린 20% 알파, 글자색 유지

투명도와 블러

  • 블러 사용 안 함. 터미널은 투명 / 블러가 아닌 불투명한 문자 스크린.
  • 오버레이 / 모달 배경: rgba(0,0,0,0.6) 블러 없음.

카드 패턴

  • 배경 #13161A, 테두리 1px solid #2A2F36, radius 2px, 그림자 없음.
  • 호버: 테두리만 #3D434D. 리프트 없음.

레이아웃 규칙

  • 최대 콘텐츠 너비: 88ch (터미널 관습)
  • 사이드바: 240–280px
  • 모든 텍스트는 왼쪽 정렬이 기본. 가운데 정렬은 히어로 타이틀에만.
  • 상단 툴바 높이: 36px (타이트). 탭 / 패널: 32px.
  • 구분선은 색이 아니라 공백 또는 ─ 라인 으로.

프로텍션 그라디언트

히어로 위 오버레이 텍스트 등에서 필요할 때만 linear-gradient(to top, #0B0D0E 0%, transparent 100%) 사용. 캡슐 / pill 로 텍스트 감싸는 것은 금지.

이미지 분위기

이미지를 쓴다면 — 채도 낮춤, 약간의 필름 그레인, 그린 틴트 (포스퍼 화면을 통해 본 것처럼). 스톡 포토 금지.

아이코노그래피 (ICONOGRAPHY)

접근 방식

유니코드 + Lucide (stroke 1.5). 이 순서대로 우선.

1. 유니코드 심볼 먼저 — 터미널 네이티브. ● ○ ◆ ◇ ▶ ▼ ✓ ✗ → ← ↑ ↓ ─ │ ╭ ╮. 가장 터미널다운 느낌.

2. Lucide (CDN, stroke 1.5px) — GUI 가 필요한 곳 (설정 버튼, 메뉴). 모노크롬.

3. 박스 드로잉 + ASCII 아트 — 장식 / 로고 용도.

4. 이모지 금지.

Lucide 를 쓰는 이유

  • Stroke 기반 (터미널 라인 드로잉과 느낌이 맞음)
  • 얇은 1.5px stroke — 무겁지 않음
  • CDN 가능, 트리셰이킹 쉬움
  • 대체 가능: Phosphor (가능하면 Regular weight)

CDN 로딩:

<script src="https://unpkg.com/lucide@latest"></script>

브랜드 로고

  • assets/logo-mark.svg — >_ 프롬프트 마크 (모노크롬, 액센트 그린 컬러 변형 가능)
  • assets/logo-wordmark.svg — kernel 워드마크 (모노스페이스, 소문자)

실제 쓰임

  • 상태: ● (running) / ○ (idle) / ✓ (success) / ✗ (error) / ◐ (partial)
  • 방향: → ← ↑ ↓
  • 리스트 불릿: ─ (대시) 또는 ·
  • 프롬프트: $ (유저), # (루트), ❯ (모던)
  • 브랜치 / 구조: ├── └──

자세한 예시는 preview/icons-unicode.html, preview/prompts-cursor.html, preview/ascii-box.html.

유의사항 (Caveats)

  • 기본 폰트 조합: JetBrains Mono (라틴 · 구글 CDN) + MaruBuri Light (한글 · 로컬). 다른 한글 폰트로 교체하려면 fonts/ 에 ttf/woff 추가 후 @font-face 의 src 와 --font-kr 값을 수정하세요.
  • 실제 제품 / 코드베이스 없음 → 커널(Kernel) 이라는 가상 CLI 도구를 가정하고 UI 키트를 구축했습니다. 실제 제품 정보를 주면 맞춰 재작업 가능합니다.
  • 한글 UI 예시 일부 포함 — 필요 없으면 제거 가능합니다.
